# Quantica — working on this project

Quantica (**quanticaedu.com**) is a problem-first math education product by solo founder
Aayan Khare. Students learn by solving guided problems with an AI tutor ("Milo") on every
problem, in the spirit of AoPS rigor delivered the way Brilliant delivers it. Free while in
early access.

## Architecture in one screen

- **`landing.html`** (~450KB, single file, no build step) is the whole front end: a
  marketing region (class-scoped `.wmb`), then the course app SPA after `<!-- SIGNUP -->`.
  It holds the app shell, rewards/XP engine, the Milo tutor UI, PSETS, auth, analytics, and
  history-based navigation. It does NOT hold lesson content.
- **`data/lessons.js`** is generated: `window.__COURSEDATA = { prealgebra:{...}, algebra1:{...} }`.
  Built by `tools/build_lessons.py` from `lessons/<course>/chapter-*.json`. Never hand-edit it.
- **`lessons/<course>/`** holds the source of truth: `chapter-N.json` (content) + `toc.json`
  (index). Courses today: `prealgebra`, `algebra1`.
- **`prealgebra/*.html`** are static SEO pages, one per lesson, generated by
  `tools/gen_seo_pages.py`.
- **`functions/`** is the Firebase Cloud Functions backend (Stripe checkout, Firestore sync).
- Milo runs on a Cloudflare Worker (`TUTOR_WORKER` in `landing.html`); its source is not in
  this repo.
- Hosting is Firebase (`firebase.json`, `cleanUrls`, `/` rewrites to `/landing.html`).

## Golden rules

**Security (standing, do not violate):**
- **Never commit, read, or touch `functions/.env`** — it holds the live Stripe secret key.
- **Deploy hosting only: `firebase deploy --only hosting`.** Never deploy functions.
- Firestore rules restrict each user to `users/{uid}/**`.

**Build + deploy loop for any content or app change:**
1. Edit `lessons/<course>/chapter-N.json` (content) or `landing.html` (app).
2. `python3 tools/build_lessons.py` (if you touched any chapter JSON).
3. Preview-test in the browser (see below); expect **0 `.katex-error`**.
4. `git commit` → `git push origin main` → `firebase deploy --only hosting`.
5. `curl` the live `data/lessons.js` or page to confirm it shipped.

**Voice (chat and content):** No em-dashes. Almost no colons in prose. Plain, natural,
AoPS-direct writing, never a poetic/abstract register. Full detail in `docs/AUTHORING.md`
and memory `no-em-dashes`.

**Answer verification:** every math answer in a lesson or PSET is machine-verified (exact
`Fraction` arithmetic, brute-force search where relevant) before it is committed. No
exceptions.

## Common tasks

- **Author a lesson** → `docs/AUTHORING.md`. Uses the Workflow template
  `tools/author_lesson.workflow.js`, then `tools/finalize_lesson.py`, then build + deploy.
- **Add a course** → `lessons/<slug>/toc.json` + `chapter-*.json`, add the slug to `COURSES`
  in `tools/build_lessons.py`, and register it in `COURSE_REG` in `landing.html`. The engine
  is course-agnostic; a course with live lessons auto-unlocks its in-app tab.
- **Regenerate SEO pages** → `python3 tools/gen_seo_pages.py <course>`.
- **Preview** → `mcp preview_start` with `.claude/launch.json` (static server on :8743), then
  open `/landing.html?course=<c>&lesson=<k>`. Note the plain static server does not apply the
  Firebase `/` rewrite, so use `/landing.html` explicitly.
